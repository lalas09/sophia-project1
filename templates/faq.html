Project Structure

I built this website using Python, HTML, and CSS. This website is basically a dictionary; the user enters a word, and the definition of that word will be returned.

There are a few different component of this application:

1. app.py - when the user enters a word, this program reads the word and returns the corresponding definition. app.py also makes sure that a word is actually entered. For example, if the user does not type in anything at all, they will get a message saying, "This did not work, please make sure your text is at least one character long". app.py also has the code for the URL. So, if the user goes to www.sophipedia.com/about, app.py makes sure that webpage corresponds to the code I have written on the about page (about.html)

2. interactive_english_dic.py - a dictionary that stores all the words and their corresponding definitions

3. home.html - This code allows the user to enter a word, and it also codes the layout of the homepage

4. faq.html - This codes the layout of the about page

5. layout.html - This is the code for the URL's (home, about, profile), and it also codes the copyright at the bottom of the page

6. main.css - This codes the layout of the pages: the color, sizes, and positions of the titles of each page
